# X Developer Portal 申請用ポリシー説明文

X Developer Portalで開発者アカウントを申請する際に必要な「開発者ポリシー」の説明文です。

## 📝 推奨説明文（250文字）

```
シビックテック井戸端キャストというポッドキャスト番組の新着エピソード情報を、RSSフィードから自動取得してXに投稿するボットを開発・運用します。配信されたエピソードのタイトル、エピソード番号、Spotifyリンクを含む告知ツイートを自動投稿し、リスナーに最新情報を提供します。投稿内容は公開情報のみを使用し、ユーザーの個人情報は一切収集・保存しません。APIは投稿機能のみを使用し、ユーザーのツイートやタイムラインへのアクセスは行いません。
```

**文字数**: 250文字（スペース含む）

## 📝 簡潔版（200文字程度）

より簡潔にしたい場合：

```
シビックテック井戸端キャストの新着エピソードをRSSフィードから自動取得し、Xに告知ツイートを投稿するボットを開発します。エピソードタイトル、番号、Spotifyリンクを含む公開情報のみを使用し、ユーザーの個人情報は収集しません。投稿機能のみを使用し、読み取り機能は使用しません。
```

**文字数**: 約200文字

## 📝 詳細版（300文字程度）

より詳しく説明したい場合：

```
シビックテック井戸端キャストというポッドキャスト番組の新着エピソード情報を、RSSフィード（Anchor.fm）から自動取得してXに投稿するボットを開発・運用します。配信されたエピソードのタイトル、エピソード番号、Spotifyリンクを含む告知ツイートを自動投稿し、リスナーに最新情報を提供します。投稿内容はRSSフィードから取得した公開情報のみを使用し、ユーザーの個人情報やプライベートデータは一切収集・保存・共有しません。X API v2の投稿機能（POST /2/tweets）のみを使用し、ユーザーのツイートやタイムラインへの読み取りアクセスは行いません。データの保存は最新エピソード番号のみで、GitHubリポジトリ内に暗号化して保存します。
```

**文字数**: 約300文字

## 🔍 各要素の説明

### 必須要素

1. **使用目的**: ポッドキャストの新着エピソードを自動投稿
2. **データソース**: RSSフィード（公開情報）
3. **使用するAPI機能**: 投稿機能のみ
4. **プライバシー**: 個人情報は収集しない
5. **データの取り扱い**: 公開情報のみ使用

### 推奨要素

- 具体的なサービス名（シビックテック井戸端キャスト）
- データソースの明示（RSSフィード、Anchor.fm）
- 使用するAPIエンドポイント（POST /2/tweets）
- データ保存方法の説明

## 📋 申請時の注意事項

1. **使用目的**: 「Making a bot」または「Exploring the API」を選択
2. **使用用途**: 上記の説明文をコピー＆ペースト
3. **アカウント情報**: 投稿に使用するXアカウントを選択
4. **利用規約**: 必ず確認して同意
